App Workflow

  • POSTING
  • Play the video.
  • Post a comment underneath.
  • An AJAX request gets sent to main service.
  • main service creates comment object with its timestamp and sends a POST request to the Node.js server.
  • The server receives a POST request at '/comments' URL.
  • The new comment object is stored in MongoDB using a Mongoose Model.
  • Success/Failure response sent back to main service.
  • If successful, latest comments are fetched via AJAX and inserted into the DOM.
  • FETCHING
  • On app load, a GET request is sent to the '/comments' URL.
  • All comments are returned to the main service in a JSON object.
  • main service sends the JSON to the Video Controller wrapped in a promise.
  • VideoCtrl resolves the promise and binds the data to the DOM.
  • DELETING
  • On clicking 'clear comments', a DELETE request is sent to the '/comments' URL.
  • Success/Failure response sent to main service.
  • HIGHLIGHTING COMMENTS
  • On playing the video, 'commentDir' directive triggers a '$watch' which which tracks video time.
  • The comments with similar timestamp to the current video time are highlighted via conditional CSS.
